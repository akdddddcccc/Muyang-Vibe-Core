# 直播贴片正式版基础

四个工作区可独立使用：背景生成、文字图层、效果融合、导出资产。

- 上贴、下贴、侧贴是独立任务；上贴成功后文字层可复用它，但文字层仍可上传自己的色彩纹理参考并独立运行。
- 字体参考默认一个、服务端去色，仅影响字形、笔画和局部纹理。
- 融合允许上传直播间底图、上贴、下贴、侧贴与文字图层；默认羽化、手绘透明遮罩、Shift 水平直线、撤销恢复属于后续切片。
- 上下贴透明边缘纹理（火焰、云层等）是羽化遮罩的后期高级边缘样式，不在第一期开放。
- 背景优先 JPEG；文字层、遮罩与合成结果使用 PNG。
- 项目配置 JSON 是后期高级能力，当前不导出。
- 共享契约已定义融合文档与图层：9:16 画板、素材引用、位置、尺寸、透明度、层级和遮罩预留字段。第一期先由浏览器 IndexedDB 本地保存，后续再迁移到 Core 项目数据服务。
- 共享契约已加入文字图层字体预设键：`elegant-songti`、`expressive-calligraphy`、`rounded-cute`、`custom-reference`。正式模型调用时，预设只作为字形参考，不能替代上贴的色彩/材质权重。
- 文字图层透明 PNG 在进入项目资产前必须按 alpha 有效像素预剪裁；Core 的未来生成管线也必须复用这一规则，避免浏览器预览和桌面/服务端导出出现文字实际尺寸不一致。

## 当前可运行边界

- `npm run dev:sticker-api` 启动本地基础服务，默认地址为 `http://127.0.0.1:8787`。
- `GET /health` 仅用于 Frontend 连接检查；返回服务版本和 Provider 是否配置，不发起模型请求。
- 项目资产持久化、生成队列、融合数据和导出任务尚未接入。
